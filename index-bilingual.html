Abstract: Promptotyping ist eine Methodologie für iteratives Prototyping mit Large Language Models. Der Entwicklungsprozess folgt dem Zyklus beschreiben, generieren, testen, verstehen. Verständnis entsteht durch das Bauen, nicht als Voraussetzung dafür.

Die Methode strukturiert die Interaktion mit Frontier-LLMs durch Simulation des Managements von Forschungs- oder Softwareprojekten. Verschiedene Modelle werden nach dokumentierten Stärken eingesetzt.

1. Einleitung

1.1 Ausgangslage

Die Integration von Large Language Models in Entwicklungsprozesse verändert Arbeitsweisen fundamental. GitHub Copilot Studien dokumentieren Produktivitätssteigerungen von 55 Prozent. Diese Adoption erfolgt jedoch ohne systematische Methodiken.

1.2 Vibe Coding als intuitiver Ansatz

Am 3. Februar 2025 prägte Andrej Karpathy den Begriff "Vibe Coding". Dieser bezeichnet eine KI-gestützte Programmierpraxis, bei der Entwickler durch vage natürlichsprachliche Anweisungen Code generieren lassen. Karpathy formuliert: "Frage dich regelmäßig ehrlich, ob du wirklich noch verstehst, was gerade passiert."

1.3 Strukturierte Alternative

Promptotyping bietet eine strukturierte Alternative zum intuitiven Vibe Coding. Die Methode kombiniert KI-assistierte Entwicklung mit systematischer Dokumentation, kontinuierlicher Validierung und expliziter Wissensexternalisierung.

2. Theoretische Grundlagen

Context Rot: LLMs werden mit zunehmender Input-Länge unzuverlässiger. Diese "Lost in the Middle" Problematik erfordert systematisches Context-Management.
Halluzinationen: Modelle generieren plausibel klingende aber falsche Details. Diese erfordern systematische Verifikation durch Domänenexperten.

Abstract: Promptotyping is a methodology for iterative prototyping with Large Language Models. The development process follows the cycle: describe, generate, test, understand. Understanding emerges through building, not as a prerequisite.

The method structures interaction with Frontier LLMs by simulating research or software project management. Different models are used according to documented strengths.

1. Introduction

1.1 Starting Point

The integration of Large Language Models into development processes fundamentally changes workflows. GitHub Copilot studies document productivity increases of 55 percent. However, this adoption occurs without systematic methodologies.

1.2 Vibe Coding as Intuitive Approach

On February 3, 2025, Andrej Karpathy coined the term "Vibe Coding". This describes an AI-supported programming practice where developers generate code through vague natural language instructions. Karpathy states: "Ask yourself regularly and honestly whether you really still understand what's happening."

1.3 Structured Alternative

Promptotyping offers a structured alternative to intuitive Vibe Coding. The method combines AI-assisted development with systematic documentation, continuous validation, and explicit knowledge externalization.

2. Theoretical Foundations

Context Rot: LLMs become less reliable with increasing input length. This "Lost in the Middle" problem requires systematic context management.
Hallucinations: Models generate plausible-sounding but false details. These require systematic verification by domain experts.

4. Critical Expert in the Loop (CEIL)

The Critical Expert in the Loop principle ensures continuous review by subject matter specialists. Unlike classical Expert-in-the-Loop, continuous intervention occurs.

Stefan-Zweig-Digital: Initially proposed multi-user system was reduced to single-page application. Result: 90% complexity reduction, 100% functionality.

5. Multi-Model Verification

Critical information undergoes standard multi-model verification. Discrepancies between different models signal systematic review needs.

Core Principle 1: The method works with the models rather than against them.
Core Principle 2: Progress emerges through iteration.
Core Principle 3: The goal is the functioning prototype as an instrument of knowledge.
