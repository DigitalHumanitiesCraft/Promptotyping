# From its first description Promptotyping set the documentation of requirements above finished software as its aim.

## Statement

The first public description sets the method's aim as documenting requirements quickly and precisely for subsequent implementation by an LLM. Producing finished software directly is not the aim; the documentation is the primary artifact of the work. These documents carry comprehensive context, among it the domain background, information on the data used, and technical formalizations.
