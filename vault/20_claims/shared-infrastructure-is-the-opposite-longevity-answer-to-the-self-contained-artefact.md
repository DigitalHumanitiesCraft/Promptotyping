# The field also answers the longevity problem the opposite way, integrating project data into shared Wikidata so a maintained collective platform carries preservation.

## Statement

Against the same limited-hosting-lifetime problem that motivates the self-contained static artefact, one strand of the field pursues the inverse answer: rather than removing dependencies, it integrates project research data directly into the Wikimedia Foundation's shared infrastructure, so that a maintained collective platform carries preservation and the data stays public after the project ends. This grounds the second half of the paper's Section 4.1 footnote precedent, where the static artefact instead reaches longevity by carrying no dependencies to maintain.
