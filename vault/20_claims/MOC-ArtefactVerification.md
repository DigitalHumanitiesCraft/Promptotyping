# MOC: ArtefactVerification

Claims behind the paper's Sections 4, 6.1, and 6.2: the self-contained static artefact type and its sustainability discourse, the interface typology and its classification gap, and the layering of verification between deterministic checks and expert judgement. Added with the bibliography-traceability sweep of 2026-07-23.

- [[20_claims/ssh-open-marketplace-models-workflows-as-step-sequences]] — The SSH Open Marketplace records research workflows as step-by-step sequences pointing to the tools, services, and datasets usable at each stage
- [[20_claims/client-side-provenance-tool-is-a-precedent-for-server-free-artefacts]] — A same-venue provenance tool runs entirely in the browser without server-side architecture, a precedent for the server-free artefact type
- [[20_claims/tadirah-most-adopted-yet-under-maintained]] — TaDiRAH is the digital humanities' most widely adopted classification scheme, yet it is under-maintained and has been dormant for years
- [[20_claims/semantic-vocabulary-matching-stays-a-human-interpretive-task]] — Semantic matching between vocabularies remains a human-driven interpretive task, and AI becomes meaningful only where grounded in curated, context-rich data
- [[20_claims/digital-edition-is-interface-gui-and-api]] — The digital scholarly edition is itself an interface, addressed as GUI for humans and API for machines
- [[20_claims/minimal-computing-resists-scale-as-innovation]] — Minimal computing is a constraint-driven mode of DH praxis that resists equating innovation with scale
- [[20_claims/digital-editing-converges-on-static-self-contained-artefact]] — Digital scholarly editing arrives from its own sustainability discourse at the self-contained static web artefact as the durable publication form
- [[20_claims/fair4rs-provenance-and-identifier-principles]] — The FAIR4RS principles require detailed provenance under Reusable (R1.2) and a persistent identifier under Findable (F1)
- [[20_claims/glam-practice-layers-llm-extraction-with-deterministic-checks-and-expert-review]] — GLAM practice brackets non-deterministic LLM extraction with deterministic regex post-checks and an expert correction interface
- [[20_claims/wikidata-tool-registries-as-commons]] — Community-curated tool registries on Wikidata answer the directory paradox by treating tool data as a maintained commons
- [[20_claims/edition-interface-embodies-editorial-decisions]] — The interface of a digital edition embodies editorial decisions and is bound to its data model
- [[20_claims/llm-vocabulary-test-2025-fell-short-of-thesaurus-interoperability]] — A 2025 test of GPT-4 and DeepL Write on specialist vocabulary lacked the terminological precision and hierarchical embedding required for thesaurus-level interoperability
- [[20_claims/minimal-computing-reduces-code-and-dependencies]] — Minimal computing means reducing computation literally: less code, fewer bytes, less processing
- [[20_claims/shared-infrastructure-is-the-opposite-longevity-answer-to-the-self-contained-artefact]] — The field also answers the longevity problem the opposite way, integrating project data into shared Wikidata so a maintained collective platform carries preservation
- [[20_claims/endings-durability-through-static-no-dependencies]] — The Endings Principles secure digital longevity by building static sites free of server-side and external dependencies
- [[20_claims/scholarly-primitives-classify-activities-not-interfaces]] — Unsworth's scholarly primitives classify research activities, not the epistemic work interfaces enable
- [[20_claims/endings-static-artefacts-minimise-maintenance]] — Endings-compliant artefacts function without JavaScript or CSS and require no ongoing tending, turning "Tamagotchis" into "Pet Rocks"
- [[20_claims/do-one-thing-well-favours-small-specialised-tools-over-monolithic-platforms]] — The Unix "do one thing well" principle, applied to digital editing, argues for small, specialised tools chained through standard formats against monolithic platforms
- [[20_claims/tadirah-classifies-dh-research-activities]] — TaDiRAH is the digital humanities' taxonomy of research activities, organising DH work under top-level research goals
- [[20_claims/coordinated-multiple-views-enable-exploration]] — Coordinated and multiple views let users understand data through different linked representations
- [[20_claims/static-client-side-editions-reach-tens-of-thousands-of-units]] — A statically published TEI edition of over 16,000 pre-rendered units processes and stores without size trouble on ordinary hardware
