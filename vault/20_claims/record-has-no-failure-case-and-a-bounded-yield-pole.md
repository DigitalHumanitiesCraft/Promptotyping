# The record carries no documented failure case, and at its negative pole stands bounded yield where the documents stayed thin.

## Statement

The project inventory and the worked cases present successes only, so a statement that failures cluster where documentation was vague has no documented case behind it. What the record does carry at the negative pole is bounded yield, the case in which a single design document is all that stands between an artefact and the visualisation conventions a model reproduces by default. A ranking of the projects by documentation depth does not carry this pole, because the verified count places the case in question at thirteen knowledge files, the same order of magnitude as the deeply documented cases, while the paper's lower figure describes an earlier build state.
