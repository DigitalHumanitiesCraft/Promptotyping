# Review is a core commitment across the Spec-Driven-Development wave, and what it examines is internal coherence, coverage and code quality.

## Statement

Every framework in the wave carries a review step and several make it an explicit obligation of the human. Spec Kit assigns verification to the person steering and runs machine-side consistency, coverage and checklist analysis; Kiro defaults to approval gates between phases and offers a requirements analysis for inconsistencies, ambiguities and gaps; BMAD runs a readiness gate before implementation and a code review after it; OpenSpec has the agent write and the human review the plan. In each case the object of the check is the artefact's own coherence and the quality of the code, and no framework binds the check to correspondence with a subject matter outside the specification.

## Support

- [[10_distillates/documents/sdd-spec-kit-2026-07-24#^s6]] — verification is an explicit and repeated obligation of the human at each phase checkpoint.
- [[10_distillates/documents/sdd-spec-kit-2026-07-24#^s8]] — machine-side assurance is cross-artefact consistency, coverage and requirement checklists.
- [[10_distillates/documents/sdd-kiro-2026-07-24#^s5]] — approval gates between the phases are the default.
- [[10_distillates/documents/sdd-kiro-2026-07-24#^s6]] — requirements checking targets internal coherence rather than correspondence to a subject matter.
- [[10_distillates/documents/sdd-bmad-method-2026-07-24#^s7]] — validation runs as a readiness gate and a code review, each returning a verdict on the state of the work.
- [[10_distillates/documents/sdd-openspec-2026-07-24#^s5]] — the agent writes and the human reviews the plan before any code is written.
