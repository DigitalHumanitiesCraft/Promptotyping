# The mandatory Promptotyping frontmatter core was reduced to the six fields the practice actually carries.

## Statement

The standardisation of Promptotyping frontmatter did not impose a rich schema but reduced the mandatory core to six fields (title, created, updated, status, project, method) after a real refactor wave, dropping the type and dependencies fields from the recommendations. The type field in particular was demoted from a mandatory field to a reading heuristic, because the refactor introduced it in no document.

## Support

- [[10_distillates/documents/frontmatter-practice-2026-05#^s2]] — records the reduction of the mandatory core to the six named fields and the removal of type and dependencies from the recommendations.
- [[10_distillates/documents/frontmatter-practice-2026-05#^s3]] — records that the refactor introduced type in no document, which is why the convention demoted it to a reading heuristic.
