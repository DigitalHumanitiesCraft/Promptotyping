# MOC: Genealogy

Claims about the method's origin and side lines: the dissertation as origin (user stories to interfaces, mapping of research data and context onto artifacts), PRISM as the side line contributing the process insight, and the dating of both.

- [[20_claims/prism-prompt-documented-by-january-2025]] — dates the PRISM side line, the prompt was documented in the public GM-DH repository by 2025-01-21.
- [[20_claims/dissertation-requirements-from-deep-dive-sessions]] — scholar-centred design captured structured requirements from deep-dive sessions with domain experts between 2017 and 2020.
- [[20_claims/dissertation-requirements-feed-built-artifacts]] — those captured requirements fed the built artifacts, interface design, TEI XML and ontology modelling, and RDF workflows.
- [[20_claims/bookkeeping-ontology-session-driven]] — session-surfaced representation needs shaped the Bookkeeping Ontology, with classes Group, Right, and AgentMention traceable to named sources.
- [[20_claims/promptotyping-named-first-described-2025-04]] — the method was named Promptotyping and first publicly described on 2025-04-24 as an iterative four-phase method.
- [[20_claims/promptotyping-documentation-over-software]] — from its first description the method set documentation of requirements above finished software (also under Method).
- [[20_claims/prism-configures-reasoning-via-parameterised-document]] — the PRISM prompt configures reasoning through a parameterised context document with rated alternatives and recursive descent.
- [[20_claims/prism-taught-summer-school-2025-09]] — PRISM featured in the Day 2 advanced-prompting session of the September 2025 LLM summer school.
