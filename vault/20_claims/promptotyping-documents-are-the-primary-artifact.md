# The Promptotyping Documents are the primary artifact; the code is a regenerable mapping.

## Statement

In the L.I.S.A. positioning, the Promptotyping Documents are distilled knowledge that keeps context explicit and are the actual artifact of the context engineering, enabling continued work even when the code is regenerated. The process focuses on one task, mapping research data onto interfaces, and what distinguishes Promptotyping from Vibe Coding is its structured procedure and active requirements analysis interwoven with documentation and implementation.
