# Distillate: BMAD-METHOD

BMAD is the role-heaviest of the Spec-Driven-Development frameworks, organising the work as a roster of agent personas that hand documents to one another across four phases. Its documentation is the strongest available test of whether the wave installs anything comparable to the Critical Expert in the Loop.

## Core statements

- The BMAD repository was created on 2025-04-13, which places it before the Spec Kit and Kiro releases of the same wave. [[00_representation/documents/sdd-bmad-method-2026-07-24#^created]] ^s1
- The framework describes itself as building context progressively across four phases whose documents inform one another. [[00_representation/documents/sdd-bmad-method-2026-07-24#^four-phases]] ^s2
- Its division of labour is a roster of specialised agents covering analysis, product management, architecture, development, user experience and technical writing. [[00_representation/documents/sdd-bmad-method-2026-07-24#^roster]] ^s3
- These roles are agent skills invoked inside the tool rather than definitions of human competence. [[00_representation/documents/sdd-bmad-method-2026-07-24#^agents-are-skills]] ^s4
- The human's stated position is that of a partner whose own thinking the facilitated workflows are meant to elicit. [[00_representation/documents/sdd-bmad-method-2026-07-24#^collaborators]] ^s5
- Its specification artefact is a five-field kernel of purpose, capabilities, constraints, non-goals and success signal, validated for preservation of load-bearing source claims. [[00_representation/documents/sdd-bmad-method-2026-07-24#^spec-kernel]] ^s6
- Validation runs as a readiness gate before implementation and a code review after it, each returning a verdict on the state of the work. [[00_representation/documents/sdd-bmad-method-2026-07-24#^gates]] ^s7
- Persistent project governance is carried by a project-context file the documentation calls a constitution for the project. [[00_representation/documents/sdd-bmad-method-2026-07-24#^project-context]] ^s8
- The documents are addressed to the agents of the following phase, which is the framework's own account of what its document chain is for. [[00_representation/documents/sdd-bmad-method-2026-07-24#^document-chain]] ^s9
- The workflow map lists no artefact whose purpose is describing external data sources or their semantics, and defines no verification role bound to subject-matter competence. [[00_representation/documents/sdd-bmad-method-2026-07-24#^negative-data-audience]] ^s10

## Terms

- **Agent persona**: in BMAD, a named role bundle installed as a skill, carrying its own menu of workflows and its own deliverables. [[00_representation/documents/sdd-bmad-method-2026-07-24#^agents-are-skills]]
- **project-context.md**: the optional governance file the documentation characterises as a constitution guiding implementation decisions across all workflows. [[00_representation/documents/sdd-bmad-method-2026-07-24#^project-context]]

## Open questions

- The Analyst persona runs domain research; whether its output ever functions as a description of the data a system will process, rather than of a market or a problem space, is not stated.
- The validation gate returns a verdict without naming the competence that licenses it.
