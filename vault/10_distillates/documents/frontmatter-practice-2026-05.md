# Distillate: Frontmatter practice across Promptotyping repositories (May 2026)

The dated May 2026 survey of the `knowledge/` folders of the active Promptotyping repositories records how heterogeneous frontmatter practice was and how the shared convention hardened from it.

## Core statements

- The May 2026 survey found frontmatter practice heterogeneous across the repositories; each repository had defined its own schema in isolation without a shared vocabulary. [[00_representation/documents/frontmatter-practice-2026-05#^heterogen]] ^s1
- After a real refactor wave the mandatory frontmatter core was reduced to six fields (title, created, updated, status, project, method), and the type and dependencies fields were dropped from the recommendations. [[00_representation/documents/frontmatter-practice-2026-05#^core6]] ^s2
- The HerData refactor introduced a type field in no document, so the convention demoted type from a mandatory field to a reading heuristic. [[00_representation/documents/frontmatter-practice-2026-05#^typedemote]] ^s3
- The convention developed from a post-hoc description of one repository into a prescriptive norm against which further repositories are measured. [[00_representation/documents/frontmatter-practice-2026-05#^norm]] ^s4

## Terms

- **Reading heuristic**: deriving a document's type from its file name rather than a mandatory `type` field, the role the convention assigns to `type` after the refactor. [[00_representation/documents/frontmatter-practice-2026-05#^typedemote]]

## Open questions

- The source leaves open how far the six-field core generalises beyond the repositories surveyed in the refactor wave.
