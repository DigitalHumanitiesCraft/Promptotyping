# Distillate: Big? Smart? Clean? Messy? Data in the Humanities (Schöch 2013)

Schöch offers a working definition of humanities data as a selectively constructed abstraction and distinguishes smart data from big data along structure and volume; knowledge/paper.md Section 2.3 draws on this framing of data and its constructedness.

## Core statements

- Data stands in for the object of study and is always a partial representation of it, not the object itself. ^s1
  > "Data is not the object of study itself, but “stands in” for it in some way. Also, data is always a partial representation of the object of study." (schoech-2013)
- Humanities data can be defined as a digital, selectively constructed, machine-actionable abstraction representing some aspects of an object of humanistic inquiry. ^s2
  > "Data in the humanities could be considered a digital, selectively constructed, machine-actionable abstraction representing some aspects of a given object of humanistic inquiry." (schoech-2013)
- Following Drucker, capturing data is not passively accepting a given but actively constructing what one is interested in. ^s3
  > "capturing data is not passively accepting what is given, but actively constructing what one is interested in." (schoech-2013)
- Schöch posits two core types of humanities data, big data and smart data, separable along structure, cleanliness, and volume. ^s4
  > "there are two core types of data in the humanities: big data and smart data." (schoech-2013)
- Smart data is structured or semi-structured, explicit, enriched, and clean, which ties it to human agency and keeps its volume small. ^s5
  > "Smart data is data that is structured or semi-structured; it is explicit and enriched, because in addition to the raw data, it contains markup, annotations and metadata. And smart data is “clean”" (schoech-2013)

## Terms

- **smart data**: structured or semi-structured, explicit, enriched, clean data, small in volume because its creation depends on human agency. [[10_distillates/publications/schoech-2013-big-smart-clean-messy#^s5]]

## Open questions

- The paper reads structured research data in established standards (TEI, RDF, CMIF) as the precondition for effective Promptotyping; Schöch's smart-data category names the same object but does not connect it to LLM processing, which is the paper's step.
