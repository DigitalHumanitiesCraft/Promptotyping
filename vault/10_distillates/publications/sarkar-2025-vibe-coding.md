# Distillate: Vibe Coding — Programming through Conversation with AI (Sarkar and Drosos 2025)

The first empirical study of vibe coding, based on think-aloud video of extended sessions; the paper cites it in Section 2.5 for the iterative goal-satisfaction cycle and the notion of material disengagement, the exploratory mode Promptotyping subsumes.

## Core statements

- The paper presents itself as the first empirical study of vibe coding, built on over eight hours of curated think-aloud video. ^s1
  > "We present the first empirical study of vibe coding. We analysed over 8 hours of curated video capturing extended vibe coding sessions with rich think-aloud reflections." (sarkar-2025, Abstract)
- Vibe coding runs as an iterative goal-satisfaction cycle alternating between prompting, rapid evaluation of generated code, and manual editing. ^s2
  > "vibe coding follows iterative goal satisfaction cycles where developers alternate between prompting AI, evaluating generated code through rapid scanning and application testing, and manual editing." (sarkar-2025, Abstract)
- The practice does not remove the need for programming expertise but redistributes it toward context management, rapid evaluation, and deciding when to switch between AI-driven and manual work. ^s3
  > "vibe coding does not eliminate the need for programming expertise but rather redistributes it toward context management, rapid code evaluation, and decisions about when to transition between AI-driven and manual manipulation of code." (sarkar-2025, Abstract)
- The authors characterise vibe coding as an early manifestation of material disengagement, orchestrating code through AI while keeping selective and strategic oversight. ^s4
  > "Vibe coding is an evolution of AI-assisted programming that represents an early manifestation of "material disengagement", wherein practitioners orchestrate code production and manipulation, mediated through AI, while maintaining selective and strategic oversight." (sarkar-2025, Abstract)
- Trust in the AI tools develops through iterative verification rather than blanket acceptance. ^s5
  > "Trust in AI tools during vibe coding is dynamic and contextual, developed through iterative verification rather than blanket acceptance." (sarkar-2025, Abstract)

## Terms

- **Material disengagement**: practitioners orchestrating code production through AI while retaining selective, strategic oversight rather than writing or reading all code directly. [[10_distillates/publications/sarkar-2025-vibe-coding#^s4]]

## Open questions

- The study describes vibe coding as a mode in its own right; the paper's claim that this mode sits inside a larger structured method is a positioning the source neither makes nor contradicts.
